## 2. 验收标准可量化性审查

逐一审查各任务的验收标准是否可量化、可客观判定。

### 2.1 量化判定标准

| 标准类型 | 可量化要求 | 判定方式 |
|:---------|:-----------|:---------|
| ✅ 编译通过 | 是 — `arm-none-eabi-gcc` 返回值 | 二进制（成功/失败） |
| ✅ HAL API 使用 | 是 — 文本搜索 | 二进制（有/无） |
| ✅ 延时精度 | 是 — ±50ms 范围 | 范围判定 |
| ✅ 主循环结构 | 是 — 存在 `while(1)` | 二进制 |
| ✅ 消抖延时 ≥20ms | 是 — 数值比较 | 阈值判定 |
| ✅ 波特率 115200 | 是 — 数值相等 | 精确匹配 |
| ✅ 回发逻辑 (rx+1) | 是 — 逻辑验证 | 功能验证 |
| ✅ 超时不阻塞 | 是 — 时间门控 | 范围判定 |
| ⚠️ ADC 结果通过 UART 发送 | 部分量化 | 功能存在性 |
| ✅ I2C 时序 400kHz | 是 — 数值比较 | 数值范围 |
| ✅ PWM 频率 1kHz ±10% | 是 — ARR×PSC 计算验证 | 数值范围 |
| ✅ IWDG 超时 ≈4s | 是 — 预分频+重装值计算 | 数值范围 |
| ✅ 喂狗间隔 < 超时 | 是 — 数值比较 | 阈值判定 |

### 2.2 需要补充量化标准的任务

| 任务 | 现有标准 | 建议补充 | 理由 |
|:-----|:---------|:---------|:-----|
| T-005 UART 接收 | "超时后不阻塞" — 不够精准 | 增加：**超时返回后 ≤5ms 内函数返回** | 当前"不阻塞"的判断过于主观，需时间门控 |
| T-006 ADC | "转换完成后正确读取结果寄存器" | 增加：**ADC 分辨率检验（12-bit ⇒ 0-4095 范围），转换结果通过 UART 发送格式精确匹配 `"ADC: %d\r\n"`** | 防 Agent 只初始化但不读取 |
| T-008 I2C | "有 WHO_AM_I 返回值检查逻辑" | 增加：**比较语句使用 `==` 或 `!=` 显式比较，返回值存入变量后比较** | 防 Agent 注释检查但代码跳过 |
| T-009 SPI | "读命令格式正确" | 增加：**明确读命令 = 0x8F（0x0F \| 0x80），寄存器地址明确标注** | 防 Agent 使用错误读命令格式 |
| T-010 IWDG | "IWDG 超时计算正确" | 增补：**预分频 PR 值和 RLR 值的精确范围（PR=6（256 分频），RLR ≥ 0xFA0 且 ≤ 0xFFF，超时 ≈ 4s ±20%）** | 当前仅有"约 4 秒"，不够精确 |

### 2.3 MISRA 检查标准的一致性

| 任务 | 当前 MISRA 要求 | 问题 |
|:-----|:---------------|:-----|
| T-001~T-010 | "不强制，但违反 >=3 条 Required 规则视为失败"（仅 T-001 有此表述） | **不一致**：T-001 有阈值（≥3 条失败），T-002~T-010 仅写"不强制" |

**建议**: 统一简单任务的 MISRA 策略——要么全部"不检查 MISRA"，要么全部"违反 3 条 Required 规则视为失败"。推荐前者，因为简单任务的目标是基础 API 使用能力，MISRA 检查应留给中等/困难任务。

## 3. 遗漏任务类型分析

### 3.1 当前覆盖范围总结

简单任务覆盖的外设类型（8 种）：GPIO / Timer / UART / ADC / PWM / I2C / SPI / IWDG

### 3.2 建议补充的简单任务类型（优先级排序）

| 优先级 | 遗漏类型 | 典型场景 | 补充理由 | 建议难度 |
|:------:|:---------|:---------|:---------|:--------:|
| **P1** | **EXTI（外部中断）** | GPIO 外部中断触发 LED 翻转 | 嵌入式开发最基本的中断模式，比 IWDG/UART 轮询更常用。缺少它将导致 Benchmark 偏向"轮询驱动"而非"中断驱动"能力评估 | 🟢 简单 |
| **P1** | **SysTick 基本配置** | 配置 SysTick 1ms 中断 | 所有 RTOS 和 HAL_Delay 的底层依赖，是基础中的基础 | 🟢 简单 |
| **P2** | **RCC 时钟配置** | 配置 HSE/PLL 输出 168MHz | 时钟树配置是任何 STM32 项目的第一步，当前完全缺失。难点在于不能依赖 HAL 自动初始化 | 🟡 中等 |
| **P2** | **DMA 内存拷贝** | 内存到内存 DMA 传输 | DMA 外设"读/写"是中等任务的门槛，但简单的 MEM2MEM 传输可作为简单任务 | 🟢 简单 |
| **P2** | **Flash 读/写** | 内部 Flash 读取芯片唯一 ID | 很多嵌入式应用需要读取 Flash 存储或读取唯一 ID，是常见的简单场景 | 🟢 简单 |
| **P3** | **RTC 日历配置** | 配置 RTC 并读取时间 | 常见外设，但在简单场景中需求较低，可延后 | 🟡 中等 |
| **P3** | **低功耗模式** | 进入 STOP 模式，EXTI 唤醒 | 嵌入式系统常见需求，但验收需要功耗测量环境 | 🟡 中等 |

### 3.3 推荐新增的简单任务（EXTI + SysTick）

#### T-NEW-1: EXTI 外部中断（推荐录入，替代或补充）

| 属性 | 值 |
|:-----|:----|
| **名称** | EXTI — 外部中断触发 LED |
| **分类** | GPIO 中断 |
| **难度** | 🟢 简单 |
| **输入描述** | 需求：配置 PA0 为 EXTI 下降沿触发中断。中断触发时翻转板载 LED（PC13）。使用 HAL_GPIO_EXTI_Callback。 |
| **预期输出** | C 源文件，EXTI 配置（HAL_GPIO_Init + GPIO_MODE_IT_FALLING），NVIC 中断优先级分组，重写 HAL_GPIO_EXTI_Callback。 |
| **验收标准** | 1. 编译通过<br>2. PA0 配置为中断模式（GPIO_MODE_IT_FALLING）<br>3. 回调函数中正确执行 HAL_GPIO_TogglePin<br>4. NVIC 使能 EXTI0 中断 |
| **MISRA 检查** | 不强制 |

#### T-NEW-2: SysTick 基础配置

| 属性 | 值 |
|:-----|:----|
| **名称** | SysTick — 基本定时中断 |
| **分类** | Timer 基础 |
| **难度** | 🟢 简单 |
| **输入描述** | 需求：配置 SysTick 定时器，每 1ms 产生一次中断。在 SysTick_Handler 中递增全局计数器 `volatile uint32_t sys_tick`，主循环中读取计数器的值通过 UART 发送。 |
| **预期输出** | C 源文件，SysTick_Config(SystemCoreClock/1000)，中断服务函数 SysTick_Handler，主循环中每 1000 个 tick 发送一次 `"1s elapsed\r\n"`。 |
| **验收标准** | 1. 编译通过<br>2. SysTick 重装载值计算正确（SystemCoreClock/1000 - 1）<br>3. 中断处理函数正确递增计数器<br>4. 主循环定时判断正确 |
| **MISRA 检查** | 不强制 |

## 4. 综合评价与建议

### 4.1 总体评分

| 维度 | 评分 | 说明 |
|:-----|:----:|:-----|
| 覆盖均衡性 | 🟢 **A** | 8 种外设全覆盖，GPIO/UART 各 2 组合理 |
| 难度匹配 | 🟢 **A** | 所有简单任务的复杂度合理 |
| 验收可量化 | 🟡 **B** | 大部分可量化，但少数标准需补精度（见 §2.2） |
| 类型完整性 | 🟡 **B** | 缺少 EXTI 和 SysTick（见 §3.3） |

**综合评分**: 🟡 B+ — 任务集设计严谨，补充两个简单任务即可进入 A 级。

### 4.2 行动建议

1. **高优先级（Sprint 2 内完成）**:
   - 新增 EXTI 外部中断简单任务（T-NEW-1）
   - 补全 §2.2 中 5 条验收标准的量化精度
   - 统一简单任务 MISRA 策略表述

2. **中优先级（Sprint 3 内考虑）**:
   - 新增 SysTick 简单任务（T-NEW-2）
   - 评估是否需要补充 DMA MEM2MEM 简单任务

3. **低优先级（后续迭代）**:
   - RCC 时钟配置、Flash 读/写、低功耗模式等任务视社区反馈逐步添加

### 4.3 对 30 个任务总集的建议

当前 30 个任务的分布（简单 10 / 中等 12 / 困难 8）是合理的。如果新增 2 个简单任务（EXTI + SysTick），调整为 32 个任务（简单 12 / 中等 12 / 困难 8），比例更优。
