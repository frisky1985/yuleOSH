# Evidence Pack Structure

> **Document**: `docs/evidence-pack-structure.md`
> **Version**: 1.0.0
> **Status**: Approved — Sprint 1

## Overview

The yuleOSH evidence pack (`audit-manifest.json` + bundled artifacts) is the
core deliverable for ASPICE SWE.1~SWE.6 audits.  Every CI pipeline run or
`yuleosh ev pack` command produces a self-contained evidence pack that
auditors can verify independently.

---

## Audit Manifest (`audit-manifest.json`)

The manifest is a JSON file generated by `evidence/manifest.py`.  Every other
file is listed in it along with its SHA-256 hash.

### Schema

| Field                      | Type       | Required | Description                                  |
|----------------------------|------------|----------|----------------------------------------------|
| `schema_version`           | string     | ✅       | Manifest schema version (e.g. "1.0.0")       |
| `build_id`                 | string     | ✅       | Build identifier (commit hash + timestamp)   |
| `generated_at`             | string     | ✅       | ISO 8601 generation timestamp                |
| `generated_by`             | string     | ✅       | Generator ID (e.g. "yuleosh-ev-cli")         |
| `evidence_pack_version`    | string     | ✅       | Semver for this pack                         |
| `files`                    | array      | ✅       | List of `ManifestFileEntry` objects          |
| `file_count`               | integer    | ✅       | Total number of files in the pack            |
| `total_size_bytes`         | integer    | ✅       | Sum of all file sizes                        |
| `sha256`                   | string     | ✅       | SHA-256 of concatenated file hashes          |
| `cross_refs_valid`         | boolean    | ✅       | Whether all cross-references resolve         |
| `unresolved_refs`          | array      | ✅       | List of unresolved references (if any)       |
| `coverage_warnings`        | array      | ❌       | Coverage reasonableness warnings             |
| `signature`                | string     | ❌       | Base64-encoded RSA-SHA256 signature          |

### ManifestFileEntry

| Field          | Type    | Required | Description                              |
|----------------|---------|----------|------------------------------------------|
| `path`         | string  | ✅       | Path relative to evidence pack root      |
| `size_bytes`   | integer | ✅       | File size in bytes                       |
| `sha256`       | string  | ✅       | SHA-256 hex digest of the file           |
| `content_type` | string  | ✅       | MIME-category ("json", "md", "zip", etc) |
| `description`  | string  | ✅       | Human-readable purpose                   |
| `required`     | boolean | ✅       | Whether the file is mandatory            |
| `cross_refs`   | array   | ❌       | IDs of other files this file references  |

## Integrity Checks

The `yuleosh evidence check` command runs 7 validation layers:

| Layer                         | What it verifies                                                         | Blocking |
|-------------------------------|--------------------------------------------------------------------------|----------|
| **files_present**             | `audit-manifest.json` + all `required: true` files exist                 | Yes      |
| **fields_complete**           | Every JSON has valid, non-empty structure                                | Yes      |
| **values_reasonable**         | Coverage > 1%, no implausible numbers                                    | Warning  |
| **timestamps_ordered**        | Pipeline step timestamps are monotonically increasing                    | Yes      |
| **cross_refs_resolved**       | Every `ref_id` in traceability has a matching file                       | Warning  |
| **sha256_integrity**          | File hashes match manifest records                                       | Yes      |
| **signature_valid**           | RSA-SHA256 signature (if present) is cryptographically valid             | Warning  |

## Digital Signature

Evidence packs can be signed with RSA-2048 + SHA-256 (PKCS1v15 padding).

### Key Management

- **Private key**: stored only in the CI/CD pipeline environment (never shipped)
- **Public key**: embedded in the yuleOSH client distribution
- **Key generation**: `python3 -c "from yuleosh.evidence.signer import generate_keypair, save_keys; save_keys(*generate_keypair())"`

### Signing Flow

1. During `yuleosh ev pack`, `generate_audit_manifest()` produces the manifest
2. If signing is enabled, `signer.sign_manifest_file()` adds a base64-encoded RSA-SHA256 signature
3. The signature is stored in `audit-manifest.json["signature"]`
4. Verification re-hashes all fields **except** `signature` and compares

## CLI Usage

```bash
# Generate evidence pack with manifest
yuleosh ev pack --build-id "abc123-20260705"

# Full multi-layer integrity check
yuleosh ev check /path/to/evidence

# Output as JSON (for automation)
yuleosh ev check /path/to/evidence --json

# Sign an existing manifest
python3 -c "
from yuleosh.evidence.signer import sign_manifest_file
sign_manifest_file('.osh/evidence/b1/audit-manifest.json', '.yuleosh/signing/private.pem')
"

# Verify signature
python3 -c "
from yuleosh.evidence.signer import verify_manifest, load_public_key
from yuleosh.evidence.manifest import load_manifest
m = load_manifest('.osh/evidence/b1/audit-manifest.json')
pub = load_public_key('.yuleosh/signing/public.pem')
import json
content = json.dumps({k:v for k,v in m.__dict__.items() if k != 'signature'}, indent=2)
print('✓' if verify_manifest(content, m.signature, pub) else '✗')
"
```
