## 1. 概述

### 1.1 为何需要升级

转角汽车电子专家在第二次复评审中指出：

> **"MISRA C:2023 已经发布。如果 yuleOSH 的 185 条规则是基于 C:2012 的，需要规划升级。C:2023 在规则分类、安全扩展集方面有重大变化。"**

MISRA C:2023（正式发布于 2023 年 12 月）是 MISRA C 标准自 2012 年以来的首次重大修订。客户审计时可能被提问：
- "你们的工具基于哪个 MISRA 版本？"
- "规则 X 在 2023 版本中有变更，你们的工具如何处理？"
- "是否支持 MISRA C:2023 新增的安全规则集？"

提前规划升级路径是 yuleOSH 保持审计可信度的必要投资。

### 1.2 yuleOSH 当前状态

| 项目 | 当前版本 |
|:-----|:---------|
| MISRA 基础标准 | MISRA C:2012 (incl. Amendment 1) |

## 2. MISRA C:2012 → C:2023 关键变化

### 2.1 新增规则

MISRA C:2023 在 C:2012 + Amendment 1 + Amendment 2（Security）基础上，新增以下规则类别：

| 类别 | 新增规则数 | 说明 |
|:-----|:----------:|:------|
| **安全扩展集（Security）** | ~40 条 | 从 MISRA C:2012 Addendum 2（2020）正式纳入核心标准 |
| **新 Required 规则** | ~15 条 | 针对现代 C 编程实践（_Generic、复合字面量、线程安全） |
| **新 Advisory 规则** | ~8 条 | 代码可读性、配置管理、头文件组织 |
| **新 Mandatory 规则** | ~5 条 | 未定义行为消除、严格别名规则增强 |
| **总计新增** | **~68 条** | |

#### 重点新增规则速览

| 规则 | 分类 | 内容 | 影响程度 |
|:-----|:----:|:-----|:--------:|
| Dir 4.14 | Mandatory | 不得依赖编译器的未定义行为优化假设 | 🔴 高 |
| Rule 1.5 | Required | `_Generic` 选择表达式不得产生副作用的参数 | 🟡 中 |
| Rule 2.8 | Required | 不得使用 `restrict` 限定的指针指向重叠对象 | 🟡 中 |
| Rule 8.18 | Required | 复合字面量不得具有非常量初始化器 | 🟡 中 |
| Rule 11.10 | Required | 指针算术不得用于指向非数组对象的指针 | 🔴 高 |
| Rule 22.7-22.12 | Required | 线程安全规则（与 C11 threads.h 相关） | 🟡 中 |
| Rule 23.1-23.8 | Security | 缓冲区溢出防护、输入验证、整数溢出检查 | 🔴 高 |

### 2.2 删除规则

以下规则在 C:2023 中被移除或合并：

| 规则 | 原分类 | 删除原因 | 替代方案 |
|:-----|:------:|:---------|:---------|
| Rule 1.1 (部分) | Required | 放松了对某些标准扩展的限制 | 允许 `__attribute__` 等常见扩展 |
| Rule 5.9 | Required | 与 Rule 5.8 语义重复 | 合并至 Rule 5.8 |
| Rule 21.6 | Required | 放宽了对标准库 `fgetpos`/`fsetpos` 的限制 | 条件允许 |
| Rule 22.1-22.5（部分） | Required | 放宽了对 `setjmp`/`longjmp` 的限制 | 仅特定场景受限 |

### 2.3 修改规则

以下规则的判定逻辑或例外条件在 C:2023 中有显著变化：

| 规则 | 变化内容 | 对 yuleOSH 的影响 |
|:-----|:---------|:-----------------|
| **Dir 4.1** | 明确扩展语法的偏差审批流程，新增偏差模板 | 偏差管理流程需更新 |
| **Rule 2.2** | 放宽对死代码的判定（允许调试宏展开的死代码） | cppcheck 策略需更新 |
| **Rule 8.13** | 允许某些嵌入式回调省略 `const` | AI Agent 审查标准需更新 |
| **Rule 10.1-10.4** | 整型转换规则细化，新增异常路径 | cppcheck misra-addon 需更新 |
| **Rule 11.3-11.5** | 指针转换规则更严格，新增 MMIO 访问例外路径 | 白名单配置需更新 |
| **Rule 14.4** | 控制表达式常量判定条件调整 | cppcheck 规则需重检 |
| **Rule 16.6** | switch 语句的 break 要求放宽（允许 fall-through with comment） | AI 审查标准需更新 |
| **Rule 17.2** | 递归禁止规则增加 tail-recursion 例外 | cppcheck 配置需更新 |
| **Rule 18.4-18.5** | 指针算术规则新增安全操作模式 | AI Agent 审查标准需更新 |
| **Rule 21.12** | 放宽对 `abort`/`exit` 的限制（允许看门狗复位场景） | 白名单配置需更新 |

### 2.4 分类体系变化

| 维度 | MISRA C:2012 | MISRA C:2023 |
|:-----|:-------------|:-------------|
| 规则分类 | Mandatory / Required / Advisory | 不变，但 Advisory 规则占比降低 |
| 方向性规则 (Dir) | 12 条 | 16 条（新增 4 条） |
| 安全扩展集 | 独立 Addendum 2 | 正式纳入核心标准 |
| 强制性等级 | Mandatory 不可偏差 | 不变 |
| 偏差管理 | 松散 | 标准化偏差模板（MISRA Compliance 2020 对齐） |
| 工具分类 | 未明确 | 明确 TCL1/TCL2/TCL3 分类 |

### 3.3 RAG 覆盖

当前 RAG 索引仅包含 30 条 C:2012 规则的详细条目。C:2023 升级后需扩展至 ~248 条规则（c.2023 全量），同时更新已有条目的规则文本。

### 3.4 AI Agent 审查覆盖

AI Agent 审查标准（prompt/checklist）基于 C:2012 编写。C:2023 的修改规则（特别是 10.1-10.4, 11.3-11.5, 18.4-18.5）需要同步更新 Agent 的审查指令。
