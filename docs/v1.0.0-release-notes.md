# yuleOSH v1.0.0 — Release Notes

> 🚀 GA Release | 2026-06-10

## What's New

### 🔥 SaaS Commercial Platform
- **Multi-tenant**: Organizations, projects, users with role-based access
- **JWT Authentication**: Signed tokens with expiry, bcrypt passwords, rate limiting
- **Stripe Payment**: Checkout sessions + webhook handling
- **Usage Metering**: Per-org pipeline runs, LLM tokens, storage tracking
- **Free Trial**: 14-day Pro trial for all new organizations

### 🚀 Developer Experience
- **Onboarding Wizard**: 4-step interactive guide (create → spec → pipeline → evidence)
- **Dashboard 2.0**: Stats overview, project cards, activity feed, quick actions
- **Async Pipeline**: Background execution with status polling API

### 🛡️ Production Ready
- **Security Headers**: CSP, X-Frame, X-Content-Type, Referrer
- **Pre-commit Guards**: Mangled env var detection, spec validation
- **Docker Compose**: Multi-service deployment with Nginx SSL

### 📊 by the Numbers
- **1,031 tests** across 36 test files
- **86%+ code coverage**
- **25 formal requirements**, 6 scenarios, 69 SHALL statements
- **14 REST API resources** with full auth
- **7 database migrations** (v1 → v7)
- **10 documentation files**

## Upgrade from v0.8.0

```bash
git pull origin main
pip install -e ".[dev]"
pip install PyJWT bcrypt stripe

# Database auto-migrates on first access
python3 src/ui/server.py
```

## Breaking Changes

- User signin now requires password for existing users with passwords set
- `_generate_token()` now requires `user_id`, `email` params (JWT migration)
