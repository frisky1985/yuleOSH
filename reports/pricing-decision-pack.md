## 2. 每项选择的影响

### 2.1 B1 选择 → 收入曲线差异

```
假设 100 个 Pro 客户（保守年增长）:
┌────────────────────────────────────────────────
│ ¥299/月:  100 × ¥3,588/年 = ¥358,800/年 ARR
│ ¥599/月:  100 × ¥7,188/年 = ¥718,800/年 ARR
│ ¥999/月:  100 × ¥11,988/年 = ¥1,198,800/年 ARR
└────────────────────────────────────────────────
```

**¥299 风险**: 不是收入少的问题。是**品牌定位**: 汽车行业采购经理看到 ¥299 → "这产品一定是个学生项目"。"便宜"在汽车行业是负面信号。

**¥999 风险**: 早期获客可能比¥299慢。但每多一个客户带来的收入和品牌价值高3.3倍。

**行业锚点**（为什么¥999合理）:

| 竞品 | 最低定价 | 单位 |
|:-----|:--------|:----|
| JetBrains All Products | ¥4,500/年 | 个人开发者工具 |
| Vector DaVinci | €10,000-30,000/年 | 独立座席 |
| LDRA TBvision | $15,000-25,000/年 | 独立座席 |
| MathWorks Simulink | ~$10,000/年 | 独立座席 |
| **yuleOSH Pro ¥9,999/年** | **¥9,999/年** | 全项目 |
| *yuleOSH ¥299/月* | *¥3,588/年* | *全项目* |

> **¥9,999/年 ≈ JetBrains 全家桶价格。** 但 yuleOSH 替代的是 Vector/LDRA（¥100K-300K/座/年），性价比碾压。定¥299会破坏这个感知。

### 2.2 B2 选择 → Enterprise 方案完整性

**做咨询包的理由**: 客户最关切的问题是"有了工具，但不会用"。ASPICE 认证需要 certified assessor，工具+人双驱动。参考 PTC/IBM 模式。

**不做咨询包的风险**: 大企业 POC 阶段无法提供差异化高级方案。对比竞争对手如果带了咨询方案，yuleOSH 缺少同等竞争力。

**注意**: B2 与 B1 不绑定。即使 B1 选了¥299, B2 也可以做 ¥298K 咨询包。

### 2.3 B3 选择 → 中国客户采购可行性

**中国重点目标企业**: 云意科技、高新兴、联合汽车电子、比亚迪、吉利、蔚来等。这些客户全部使用人民币采购。如果不能走人民币+增值税专票+年度合同，大部分流程阻塞在第一关。

**不做合规的风险评分**: 🔴 高。至少 80% 的中国潜在客户将无法走采购流程。

## 3. 老陈建议 ¥999 的理由（原文还原）

老陈在五轮审查中传递的核心逻辑:

> **"汽车行业的企业级工具，定价就是品牌。你定¥299，客户的第一反应是'这工具肯定是学生/个人项目，我们不敢用'。"**
>
> **"你看 Vector 一个 DaVinci 座席 €10K-30K/年，LDRA $15K-25K/年。凭什么叫这么贵？因为客户认。你一个覆盖全链条的 AI 平台，定 ¥999/月，已经是他们的 1/10。"**
>
> **"开源版免费引流，Pro ¥999 做主力收入，Enterprise ¥98K+¥298K 做大单。三层分明。"**

**核心洞察**: 老陈的逻辑不是"越贵越好"，而是"定价必须传递品牌信号"。¥299 在个人 SaaS 领域很正常（比如 Notion ¥10/月），但在企业嵌入式工具领域，"便宜"意味着"不专业"。
