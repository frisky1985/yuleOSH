- 17 条 BP 各有预定义的修复指引（`_FIX_GUIDANCE` 字典）

## 3. G-09 — fail_on_violation → fail_on_required 默认值

### 概要
将 MISRA 违规阻断默认值从 `fail_on_violation=True`（阻断所有违规）改为 `fail_on_required=True`（仅阻断 Required 违规）。Advisory 违规默认不阻断，可通过 `fail_on_advisory=True` 单独开启。

### 修改文件
| 文件 | 修改内容 |
|:-----|:---------|
| `src/yuleosh/ci/config.py` | `MisraConfig`: 新增 `fail_on_required: bool = True`, `fail_on_violation` 默认改为 `False` |
| `src/yuleosh/ci/stages.py` | 阻断逻辑: Required 检查 `fail_on_required`（默认 True），`fail_on_violation` 降级为遗留总开关（默认 False） |
| `src/yuleosh/ci/yaml_validator.py` | Schema 新增 `fail_on_required` 字段校验 |
| `.yuleosh/ci-config.yaml` | 更新 MISRA 配置段，添加注释说明 |
| `specs/misra-acceptance-matrix.md` | 更新默认值和字段说明 |
| `tests/test_compliance.py` | 更新测试断言匹配新默认值 |

### 验收标准
| # | 验收项 | 状态 |
|:-:|:-------|:----:|
| §G09.1 | `MisraConfig.fail_on_required` 默认 `True` | ✅ |
| §G09.2 | Required 违规默认阻断 pipeline | ✅ 阻断逻辑在 stages.py 795-798 |
| §G09.3 | `fail_on_violation` 默认改为 `False` | ✅ 保留为遗留主开关 |
| §G09.4 | 向下兼容: 显式设置 `fail_on_violation=True` 仍生效 | ✅ 阻断逻辑独立检查 |
| §G09.5 | Advisory 违规默认不阻断 | ✅ `fail_on_advisory` 默认 `False` |
| §G09.6 | ci-config.yaml 模板已更新 | ✅ |
| §G09.7 | yaml_validator schema 已更新 | ✅ |
| §G09.8 | 现有测试全部通过 | ✅ 167 passed |

### 阻断逻辑层级（修正后）
```
1. fail_on_required (default True)  → Required 违规阻断 pipeline
2. fail_on_violation (default False) → 遗留: 任意违规阻断（与 Required 叠加）
3. fail_threshold (default 10)      → 违规总数 ≥ N 阻断
4. violations_per_kloc (default 2.0)→ 密度超标阻断
5. fail_on_advisory (default False) → Advisory 违规阻断
```
