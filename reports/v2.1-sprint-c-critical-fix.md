## C1: TM-02 — 追溯矩阵覆盖修复

### 根因
`traceability.py` 的 `extract_shall_statements()` 存在两个问题：
1. 未从章节标题（`### RS-001:`、`#### SWR-001.1:`）提取 req_id，导致所有 SHALL 语句无 spec-defined ID
2. `generate_lrm()` 用 `SHALL-N` 格式 ID（而非 `RS-001` 格式）检索测试，无法匹配 pytest 文件中的 req_id 引用

### 修复内容
- 在 `extract_shall_statements()` 中添加 `section_req_id_pattern` 正则匹配章节标题中的 req_id
- 过滤 GIVEN/WHEN/THEN 场景块的 SHALL 语句（排除文档伪需求）
- 添加 `_scan_test_pytest_files()` 函数，直接扫描 pytest 文件中的 req_id 引用
- 添加 `_find_code_by_keywords_for_id()` 按 spec req_id 扫描源文件
- `generate_lrm()` 改用 `shall["req_id"]`（如 `RS-001`）而非 `shall["id"]`（`SHALL-N`）检索测试映射

### 效果
| 指标 | 修复前 | 修复后 |
|:-----|:------:|:------:|
| 提取 SHALL 数 | 244 | 184 |
| 有测试映射 | 0 | 36 |
| missing_test_count | 244 | 148 |

## C3: MP-03/MP-04 — Timezone Bug

### 根因
存储的 JSONL 数据中的时间戳有时区偏移（如 `2026-06-15 00:16:09 +0800`），而代码中用 `datetime.now()`（naive）进行比较。`datetime.fromisoformat()` 返回时区感知对象，导致 `TypeError: can't compare offset-naive and offset-aware datetimes`。

### 修复内容
在 `_parse_ts()` 和 `_parse_timestamp()` 中添加 `.replace(tzinfo=None)`，统一为 naive datetime 比较。

### 受影响命令
- `yuleosh coverage trend` 🛠️✅
- `yuleosh kpi status` 🛠️✅

### 文件变更
- `src/yuleosh/ci/kpi.py` — `_parse_ts()` 函数
- `src/yuleosh/ci/coverage_trend.py` — `_parse_timestamp()` 函数

## C6: TM-05 — MISRA Deviate Create CLI

### 根因
`yuleosh_cli.py` 的 `cmd_misra_deviate()` 只有 `list`/`approve`/`reject`/`add`（交互式）子命令。缺少非交互式的 `create` 命令，无法在 CI 脚本中自动创建偏差记录。

### 修复内容
1. **新增 argparse 子命令**: `misra deviate create`
   - 位置参数: `<rule>`（规则ID）、`<file>`（文件模式）
   - 可选参数: `--reason`、`--approved-by`、`--expires`、`--status`
2. **新增 `_cli_add_deviation()` 函数**: 非交互式写入 ci-config.yaml

### 使用示例
```bash
# 创建
$ python3 yuleosh_cli.py misra deviate create Rule-8.1 src/**/gen/*.c \
    --reason "Generated code" --approved-by architect --expires 2027-01-01 --status open
✅ Deviation created: Rule-8.1:src/**/gen/*.c (status: open)

# 审批
$ python3 yuleosh_cli.py misra deviate approve Rule-8.1
✅ Deviation Rule-8.1:src/**/gen/*.c → APPROVED

# 驳回
$ python3 yuleosh_cli.py misra deviate reject Rule-8.1
✅ Deviation Rule-8.1:src/**/gen/*.c → REJECTED

# 列表
$ python3 yuleosh_cli.py misra deviate list
→ 显示所有偏差记录
```

### 文件变更
- `yuleosh_cli.py` — `cmd_misra_deviate()` + `_cli_add_deviation()` 新函数 + argparse
