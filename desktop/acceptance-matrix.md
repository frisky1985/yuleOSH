## 验收矩阵

每条 SHALL 条款的通过条件、测试策略和优先级。

### 字段说明

- **P0**：MVP 阻断项 — 不通过则不可发布
- **P1**：重要缺陷 — 需修复，个别场景可带已知 workaround 发布
- **P2**：体验优化 — 低优先级
- **UT**：单元测试
- **IT**：集成测试
- **MT**：手动测试
- **E2E**：端到端自动化测试

### 模块 2：内嵌 Python 后端

| ID | SHALL 条款 | 通过条件（Pass Criteria） | 优先级 | 测试策略 |
|---|---|---|---|---|
| **AC-2.1.1** | main 进程 spawn Python 后端子进程 | `child_process.spawn` 被调用，参数包含 `yuleosh server`（或等效命令） | **P0** | UT（mock spawn 验证调用）+ IT（实际 spawn 验证进程存活） |
| **AC-2.1.2** | 启动命令为 `yuleosh server` | spawn 的 command 字符串为 `yuleosh`，args 包含 `server` | **P0** | UT |

| **AC-2.1.4** | 健康检查通过后通知渲染进程 | 健康检查成功后，主进程通过 IPC 发送 `backend-ready` 事件给渲染进程 | **P0** | UT（mock IPC 验证事件发送） |

| **AC-2.2.1** | 渲染进程直接调用 API | 渲染进程通过 `fetch('http://localhost:18788/api/v1/...')` 直接调用 Python REST API，响应正确返回 | **P0** | IT（启动后端，发送测试请求，验证响应 + CORS 头） |
| **AC-2.2.2** | Python 后端配置 CORS | Python 后端响应中包含正确的 CORS 头（`Access-Control-Allow-Origin: *` 或匹配的来源），跨域请求不被浏览器拦截 | **P0** | IT（发送跨域请求，验证 CORS 头）+ MT（前端实际调用验证） |
| **AC-2.3.1** | 构建测试验证后端可启动 | CI 构建流程包含一个测试步骤：spawn Python 后端，健康检查端点返回 200 后退出 | **P0** | CI 流水线测试（自动化） |

## 测试优先级速查

| 测试类型 | 适用场景 | 建议工具 |
|---|---|---|
| 单元测试（UT） | 进程管理逻辑、IPC 事件、菜单构造、错误处理逻辑 | Jest + electron-mock-ipc |
| 集成测试（IT） | 真实 spawn 后端 + 健康检查 + API 代理、构建产物内容 | Jest / Mocha |
| 手动测试（MT） | 窗口行为、系统托盘、原生菜单外观、启动时间、包安装 | 人工 |
| CI 自动化 | 构建编译通过、体积检查、后端可启动性 | GitHub Actions + self-hosted runner |
